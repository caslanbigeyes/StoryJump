你现在这个问题，其实已经不是“生图问题”了。

而是：

> “旁白驱动逻辑”和“镜头驱动逻辑”完全脱节。

你描述的现象：

```text id="dclhfr"
每个镜头都像独立短视频
配音像在重复讲同一件事
没有时间推进
没有事件推进
没有镜头节奏
```

本质原因是：

# 你的系统现在是：

```text id="wlq4sw"
全文案
↓
切成 N 个镜头
↓
每个镜头独立生成 prompt
↓
同一段旁白套所有镜头
```

所以：

```text id="m02l7u"
镜头 ≠ 叙事
```

只是：

> “图片集合 + 一段配音”。

这就是为什么会像 PPT。

---

# 真正的问题核心

你现在缺的是：

# Narrative Timeline（叙事时间线）

也就是：

```text id="fvwx57"
故事在推进
```

而不是：

```text id="ew8u7h"
同一个情绪重复播放
```

---

# 为什么会“每个分镜都一样”？

因为 AI 会偷懒。

例如：

你给：

```text id="24hf9l"
长安大战
```

然后生成 20 个镜头。

模型会：

```text id="d7a53q"
李白站着
李白挥剑
李白回头
李白在火里
李白看远方
```

因为：

> 模型不知道“剧情推进”。

它只知道：

```text id="1hvvh5"
继续生成相关画面
```

但电影真正需要：

```text id="w1i4gc"
事件变化
```

---

# 电影感来自什么？

不是“好看的图”。

而是：

# 因果推进

例如：

```text id="p6x9yy"
发现敌人
↓
拔剑
↓
冲锋
↓
被压制
↓
反击
↓
终结
```

这叫：

# Beat（剧情节拍）

而你现在没有 Beat。

---

# 你现在最大的结构问题

你现在的：

```text id="1h6ifv"
shot
```

其实只是：

```text id="7t5pqm"
图片节点
```

不是：

```text id="gj1xx9"
剧情节点
```

---

# 正确结构应该是：

# Story Beat → Shot

例如：

```json id="2z9b4e"
[
  {
    "beat": "魔种攻破城门",
    "purpose": "建立危机"
  },
  {
    "beat": "百姓逃亡",
    "purpose": "建立压迫感"
  },
  {
    "beat": "李白登场",
    "purpose": "英雄亮相"
  },
  {
    "beat": "第一次交锋失败",
    "purpose": "制造挫败"
  },
  {
    "beat": "铠开启魔铠",
    "purpose": "高潮"
  }
]
```

然后：

# 每个 Beat 再拆镜头

例如：

```text id="f9gz54"
Beat：李白登场

shot1：
远景，雨夜长街

shot2：
脚步踩水

shot3：
剑出鞘特写

shot4：
低机位人物亮相
```

这样：

> 镜头才是“服务剧情”。

---

# 你现在的配音为什么重复？

因为：

## 你现在的 narration 很可能是：

```text id="ckit4m"
一大段完整旁白
```

然后：

```text id="x1v52q"
每个镜头随机切一部分
```

这会导致：

```text id="mqfgzn"
每个镜头情绪差不多
```

所以：

# 必须 Beat 级 narration

例如：

```json id="i7v1fd"
[
  {
    "beat": "城门失守",
    "narration": "长安的火焰，照亮了整个黑夜。"
  },
  {
    "beat": "百姓逃亡",
    "narration": "尖叫声与马蹄声混杂在暴雨里。"
  },
  {
    "beat": "李白登场",
    "narration": "就在所有人绝望时，一道剑光划破长街。"
  }
]
```

注意：

## 每段旁白必须：

```text id="2q3bca"
推动剧情
```

而不是：

```text id="i3bm7r"
重复描述气氛
```

---

# 真正的视频叙事结构

你应该变成：

```text id="69e75c"
Story
↓
Beat（剧情节拍）
↓
Scene（场景）
↓
Shot（镜头）
↓
Frame（画面）
```

你现在直接：

```text id="96u7y0"
Story
↓
Shot
```

中间缺了两层。

---

# 你现在最应该立刻做的改动

# 第一：加入 Beat Planner

这是最关键的。

例如：

```json id="j0kq7m"
{
  "beat_id": 1,
  "goal": "建立危机",
  "emotion": "压迫",
  "event": "魔种冲入长安",
  "duration": 6
}
```

---

# 第二：镜头必须服务 Beat

不要：

```text id="rnl2k5"
随机生成帅图
```

而是：

```text id="cc5rwp"
镜头是剧情工具
```

---

# 第三：旁白不要全局生成

必须：

# 每个 Beat 独立 narration

例如：

```text id="n3c7ei"
Beat 1：
建立世界

Beat 2：
制造危机

Beat 3：
英雄登场

Beat 4：
高潮战斗

Beat 5：
结尾释放
```

---

# 第四：建立镜头节奏模板

例如：

## 开场：

```text id="d3p6e4"
慢
广角
建立环境
```

## 战斗：

```text id="v74w03"
快切
近景
运动镜头
```

## 结尾：

```text id="2vdh0f"
慢镜头
情绪释放
```
